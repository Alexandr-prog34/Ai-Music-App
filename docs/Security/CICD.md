
> CI/CD — это набор DevOps‑практик, автоматизирующих сборку, тестирование и выпуск программного обеспечения.

CI/CD позволяет разработчикам часто вносить изменения в код, который автоматически проходит сборку, тестирование и подготовку к релизу. Это обеспечивает стабильность системы, быстрое выявление ошибок и надёжную доставку продукта пользователям.

---

## 📌 Основные компоненты CI/CD

### 1️⃣ Continuous Integration (CI — Непрерывная интеграция)

Разработчики регулярно объединяют (merge) изменения кода в общую ветку.

После каждого изменения система автоматически:

- собирает приложение
    
- запускает тесты
    
- проверяет корректность сборки

Это помогает:

- избегать конфликтов версий
    
- находить ошибки на раннем этапе
    
- поддерживать код в рабочем состоянии

---

### 2️⃣ Continuous Delivery (CD — Непрерывная доставка)

После успешного прохождения тестов код автоматически разворачивается в тестовую среду (staging).

Выпуск в продакшн может происходить:

- вручную (по кнопке)
    
- после подтверждения
    

Цель — сделать релиз максимально предсказуемым и безопасным.

---

### 3️⃣ Continuous Deployment (CD — Непрерывное развертывание)

Более автоматизированная стадия.

Каждое изменение, прошедшее тестирование, автоматически публикуется в продакшн без участия человека.

Это позволяет выпускать обновления очень часто — вплоть до нескольких раз в день.

---

## 🚀 Преимущества использования CI/CD

### 🔹 Скорость

- Быстрая доставка новых функций
    
- Частые релизы (ежедневные или даже чаще)
### 🔹 Качество

- Автоматизированные тесты
    
- Быстрое обнаружение ошибок
    
- Повышение стабильности кода
### 🔹 Надёжность

- Минимизация ручных операций
    
- Устранение человеческого фактора
    
- Повторяемый и предсказуемый процесс релиза

## 🛠 Популярные инструменты CI/CD

- Jenkins — гибкий open‑source сервер автоматизации
    
- GitLab CI/CD — встроен в GitLab
    
- GitHub Actions — интегрирован в GitHub
    
- CircleCI — облачная платформа автоматизации
    

---

## 🔄 Что такое Pipeline

CI/CD формирует так называемый «конвейер» (pipeline) — последовательность автоматических этапов, через которые проходит код:

1. Коммит
    
2. Сборка
    
3. Тестирование
    
4. Развертывание
    
5. Продакшн

Код движется по этому конвейеру автоматически — от написания до выпуска.
