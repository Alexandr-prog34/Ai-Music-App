# A01 — Broken Access Control

## Что это

Нарушение контроля доступа. Пользователь получает доступ к ресурсам, которые ему не принадлежат.

## Как может проявиться в нашем проекте

- Получение чужого `job`
- Скачивание чужого трека
- Изменение статуса чужой задачи
- Подключение к WebSocket по чужому `jobID`
- Прямой доступ к `/jobs/{id}` без проверки владельца

## Почему это критично

Наш сервис хранит пользовательские данные и генерирует приватный контент.  
Любая ошибка проверки `userID` = утечка данных.

# A02 — Cryptographic Failures

## Что это

Неправильное использование криптографии или её отсутствие.

## Возможные ошибки

- JWT без проверки подписи
- Слабый секрет
- Отсутствие expiration
- Логи с токенами
- Хранение API-ключа в коде
- HTTP вместо HTTPS

## Риск

Компрометация аккаунтов и API-ключей.

# A03 — Injection

## Что это

Внедрение вредоносных данных в систему.

## Где возможно у нас

- SQL injection
- Redis injection
- Log injection
- Command injection
- JSON injection

## Примеры зон риска

- Поиск job по ID
- Фильтры
- Redis ключи
- Вызов внешних процессов
- Логирование пользовательских данных

# A04 — Insecure Design

## Что это

Архитектурные ошибки безопасности.

## Возможные проблемы

- Нет rate limiting
- Нет ограничения числа job на пользователя
- Нет ограничения длины prompt
- Нет ограничения размера запроса
- Нет ограничения WebSocket подключений

## Риск

DDoS, злоупотребление API, перегрузка Suno.

---

# A05 — Security Misconfiguration

## Что это

Неправильная конфигурация окружения.

## Возможные ошибки

- Redis открыт наружу
- Docker контейнеры с лишними портами
- DEBUG режим в продакшене
- Stack trace в HTTP ответе
- Отсутствие CORS политики
- Публичный `.env`

# A07 — Identification and Authentication Failures

## Что это

Ошибки аутентификации.

## Примеры

- JWT без expiration
- Нет проверки подписи
- Нет refresh token механизма
- Нет blacklist токенов
- Нет проверки revoked токенов
- Можно использовать токен после logout

## Риск

Захват аккаунтов.

---

# A08 — Software and Data Integrity Failures

## Что это

Нарушение целостности кода или данных.

## Возможные проблемы

- Webhook от Suno не проверяется
- Нет проверки подписи callback
- Нет проверки источника запроса
- Docker image без фиксации версии
- Нет проверки хеша скачанных файлов

---

# A09 — Security Logging and Monitoring Failures

## Что это

Отсутствие логирования атак и мониторинга.

## Возможные проблемы

- Не логируются неудачные попытки входа
- Не логируются 403
- Нет мониторинга частоты запросов
- Нет alert'ов
- Нет отслеживания WebSocket abuse

---

# A10 — Server-Side Request Forgery (SSRF)

## Что это

Сервер выполняет HTTP-запрос по адресу, который передал пользователь.

## Как может возникнуть

- Пользователь передаёт URL для загрузки аудио
- Backend скачивает файл
- Возможен доступ к внутренним сервисам
- Доступ к 169.254.169.254
- Доступ к Redis или Docker API

## Риск

Компрометация инфраструктуры.
